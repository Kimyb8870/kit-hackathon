### 리서치 완료 — 4개 플랫폼 주요 발견

**교차 분석 결과 — 공통 페인포인트 5대 축:**
1. 콘텐츠 품질 관리 체계 부재 (모든 플랫폼 TOP)
2. 환불/결제 불투명성
3. 학습 관리 도구 부족 (노트, 복습, 개인화 경로)

5. 수료증 실질 가치 없음

### 피봇 — "AI 강의 코치" → "나만의 AI 튜터" 리브랜딩 + 기능 재정의

사용자 피드백을 반영하여 컨셉 전환. "강사의 분신/코치" 프레이밍에서 **"나만의 AI 튜터"**(개인화 1:1 튜터 경험 강조)로 리브랜딩.

**새로운 4대 핵심 기능:**

| # | 기능 | 설명 |
|---|------|------|
| 1 | 커리어 상담 기반 강의 추천 | AI 튜터가 사전조사(희망 직무, 일일 학습 가능 시간 등) → 개인 최적 강의 추천 |
| 2 | 망각곡선 기반 복습 시스템 | 구매 강의 기반 최적 학습 타임라인 + 간격 반복(spaced repetition) 복습 |
| 3 | 강의 내용 Q&A | 모르는 내용 질문 → 관련 강의 영상(개별 클립 단위)까지 매칭하여 안내 |
| 4 | 개인화 튜터링 | 학습자 프로필(직무 목표, 가용 시간, 학습 이력) 기반 맞춤 코칭 |

**기술 아키텍처:** LangGraph + Tool Calling 기반 에이전트 구조 — AI 튜터가 상황에 따라 적절한 도구(강의 검색, 복습 스케줄링, 콘텐츠 Q&A 등)를 자율 호출.

**기존 5대 기능**(RAG 튜터, 코드리뷰, Q&A, 대시보드, 포트폴리오) **→ 새 4대 기능으로 PRD 리라이트 필요.**

- 학습 일정 관리 + 커리큘럼 인식 코드 리뷰 = 시장에 없는 기능

### [CRITICAL] End-to-End UX 흐름 — 단계 간 연결고리 부재

사용자가 전체 UX 흐름의 "단계 간 연결고리 부재"를 **critical**로 지적.

**핵심 문제:** 온보딩 → 첫 추천 → 강의 링크 → 수강 신청 → instructor 흐름까지 이어지는 연결 고리가 전혀 없음. 각 단계가 고립되어 있어 신규 사용자가 첫 칩("내가 등록한 강의 진행 상황")에서 즉시 dead end.

**끊어진 고리 7개:**
1. 온보딩 완료 후 자동 강의 추천 없음 (정적 환영만)
2. 예시 칩이 신규/기존 사용자 구분 없이 동일
3. AI 추천 응답에 LMS 강의 링크 미포함
4. MessageBubble에 `<a>` 컴포넌트 미정의 (마크다운 링크 스타일 없음)
5. System prompt에 링크 생성 지시 없음
6. 수강 신청 후 instructor 흐름 자동 진입 없음
7. 데모 시나리오 3 Step 간 데이터 연결 없음

### E2E 연결고리 수정 착수 — 7건 수정 + instructor→platform 파악 병렬

사용자가 7개 끊어진 고리 수정 승인. 추가로 instructor→platform 연결고리도 파악 요청. 두 작업을 병렬 진행.

**병렬 작업 구조:**
- **수정 트랙:** Learner 흐름 7개 끊어진 고리 수정 (온보딩→추천→링크→수강→instructor)
- **파악 트랙:** instructor→platform 흐름의 추가 끊어진 고리 조사

**파악 트랙 결과 — Instructor→Platform 4건 추가 끊김 발견:**
1. System Prompt에 cross-agent 협업 지시 부재
2. **Platform tools가 instructor 이벤트를 전혀 조회 안 함 (LEARNER만 조회)** — 가장 심각
3. Instructor 예시 칩에 Platform 연결 유도 없음
4. 강사가 분석 결과의 Platform 도달 여부 확인 불가

**총 끊어진 고리: Learner→Instructor 7곳 + Instructor→Platform 4곳 = 11곳**

## 핵심 결정사항

1. **문제 정의 선행 접근 채택** — 공모전 주제("AI활용 차세대 교육 솔루션")에 대해, 기존 교육 플랫폼의 실제 사용자 페인포인트를 먼저 수집하여 문제를 정의한 후 솔루션을 설계하기로 결정. 이유: 심사 가이드라인이 단순 LMS 구축이 아닌 실질적 문제 해결을 요구하므로, 근거 기반 기획이 필요.

2. **4개 플랫폼 공통 페인포인트 5대 축 확정** — AI로 해결 가능한 교차 문제를 식별하기 위해 공통점 중심으로 정리:
   - 콘텐츠 품질 관리 체계 부재 (모든 플랫폼 공통 TOP 이슈)
   - 환불/결제 불투명성
   - 학습 관리 도구 부족 (노트, 복습, 개인화 경로)
   - 강사-수강생 소통 단절
   - 수료증 실질 가치 없음

3. **Council 합의 — AI 튜터/멘토 방향 확정** — Claude("AI 강의 어시스턴트")와 Codex("AI 실습 코치 루프") 양측 분석을 종합하여, AI 튜터/멘토를 1순위 방향으로 수렴. 합의 조건: 코드 리뷰 기능 포함, 1주 MVP 범위 제한, KEG 비즈니스 적합성 중시, "강사 보조" 프레이밍. 이유: 두 모델 모두 "강사-수강생 소통 단절" 페인포인트를 AI로 해결하는 것이 공모전 심사 기준과 KEG 사업 적합성 양면에서 최적이라고 판단.

4. **공모전 주제 최종 확정: "AI 강의 코치"** — 강사의 분신이 수강생 실습을 코칭 + 학습 여정을 포트폴리오로 전환하는 시스템. Council 종합 추천안 그대로 채택. 이유: Claude의 완성도 우선 전략(깊이)과 Codex의 포트폴리오 전환 차별점(넓이)을 결합한 것이 500팀 규모 공모전에서 최적의 경쟁력을 가진다고 판단.

5. **차별점 2개 시장 검증 완료** — 경쟁사 매트릭스에서 "커리큘럼 인식 코드 리뷰"와 "학습→포트폴리오 자동 전환" 두 기능 모두 시장에 존재하지 않음을 확인. 이유: 경쟁 분석 매트릭스의 빈 공간(blue ocean)으로 확인되어, AI 강의 코치의 핵심 차별점으로 확정.

6. **발표 전략 확정 — 차별점 전면 배치** — AI 튜터(보편적, 레드오션)보다 "커리큘럼 인식 코드 리뷰"와 "포트폴리오 전환"을 발표 전면에 내세우기로 결정. 이유: 해커톤 수상작 30개+ 분석에서 이 두 기능이 시장에 없음이 3차 재확인됨. AI 튜터를 앞세우면 ChatEDU 등 기존 수상작과 동일 인상을 줄 위험.

7. **리브랜딩: "AI 강의 코치" → "나만의 AI 튜터"** — 사용자 관점에서 개인화된 1:1 튜터 경험을 강조하기 위해 명칭 및 컨셉 전환. 이유: "코치/강사 분신"은 공급자(강사) 관점이었으나, 수요자(학습자) 관점의 네이밍이 공모전 심사에서 더 직관적.

8. **기술 아키텍처: LangGraph + Tool Calling 에이전트 구조 확정** — AI 튜터가 상황에 따라 적절한 도구(강의 검색, 복습 스케줄링, 콘텐츠 Q&A 등)를 자율 호출하는 구조. 이유: 단일 챗봇이 아닌 다기능 에이전트로서의 튜터를 구현하기 위해 LangGraph의 상태 기계 + Tool Calling 조합이 최적.
